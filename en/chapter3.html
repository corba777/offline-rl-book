Chapter 3: Conservative Q-Learning (CQL)

"Don't trust values you haven't seen. And if you haven't seen them, push them down."

The Problem, Restated

In Chapter 2 we saw that standard Q-learning on offline data produces catastrophically overoptimistic Q-values for out-of-distribution (OOD) actions. The greedy policy then exploits these inflated values, selecting actions the dataset never covered — and failing in deployment.

Conservative Q-Learning (CQL) — Kumar et al., NeurIPS 2020 — fixes this with one elegant idea: add a regularization term that explicitly penalizes Q-values on OOD actions while preserving them on dataset actions.

The result: a Q-function that is pessimistic about OOD actions by construction. The greedy policy, facing lower values outside the dataset, naturally stays close to the behavior policy — without explicitly constraining it.

The Idea

Standard TD training minimizes:

$$\mathcal{L}_{TD}(\theta) = \mathbb{E}_{(s,a,r,s') \sim \mathcal{D}} \left[ \left( r + \gamma \max_{a'} Q_{\bar\theta}(s', a') - Q_\theta(s, a) \right)^2 \right]$$

CQL adds a conservative penalty:

$$\mathcal{L}_{CQL}(\theta) = \mathcal{L}_{TD}(\theta) + \alpha \cdot \underbrace{\left( \mathbb{E}_{s \sim \mathcal{D},\, a \sim \mu} \left[ Q_\theta(s, a) \right] - \mathbb{E}_{(s,a) \sim \mathcal{D}} \left[ Q_\theta(s, a) \right] \right)}_{\text{push OOD down, push dataset up}}$$

where $\mu$ is a proposal distribution over actions (uniform or current policy), and $\alpha > 0$ controls conservatism strength.

In words: minimize Q everywhere, but maximize at dataset points. The gap between the two expectations is what CQL penalizes.

Formalization

The Full Objective

CQL minimizes a regularized Bellman error:

$$\min_\theta \, \alpha \left( \mathbb{E}_{s \sim \mathcal{D}} \left[ \log \sum_a \exp Q_\theta(s, a) \right] - \mathbb{E}_{(s,a) \sim \mathcal{D}} \left[ Q_\theta(s, a) \right] \right) + \frac{1}{2} \mathcal{L}_{TD}(\theta)$$

The log-sum-exp term is a smooth approximation of $\max_a Q(s,a)$ — it pushes the entire Q-surface down. The second term lifts Q-values specifically at dataset points.

For continuous action spaces (like our thermal process), enumerating all $a$ is impossible. CQL approximates logsumexp by importance sampling with two proposal distributions — random uniform actions and current policy actions:

$$\log \sum_a \exp Q(s, a) \approx \log \mathbb{E}_{a \sim \mu} \left[ \frac{\exp Q(s,a)}{\mu(a|s)} \right]$$

This is exactly what compute_cql_loss() does in cql.py.

Theoretical Guarantee

Theorem (Kumar et al., 2020). Under appropriate $\alpha$, the Q-function learned by CQL satisfies:

$$\hat{Q}^\pi(s, a) \leq Q^\pi(s, a) \quad \forall (s, a) \in \mathcal{D}$$

CQL is a lower bound on the true Q-function at dataset points. The policy trained on this pessimistic estimate cannot exploit overestimated values — because they don't exist.

Environment: ThermalProcessEnv

The toy environment mimics an asphalt coating process. State is [temperature, filler_pct, viscosity], all normalized to [0, 1]. Actions [heat_delta, flow_delta] are in [-1, 1]. Dynamics are first-order with additive noise:

class ThermalProcessEnv:
    def __init__(self, T_target=0.6, f_target=0.5, max_steps=200,
                 noise_std=0.02, seed=42):
        self.alpha_T, self.beta_T = 0.92, 0.08
        self.alpha_f, self.beta_f = 0.88, 0.12

    def _viscosity(self, T, f):
        """Nonlinear: high temp -> low viscosity; more filler -> high viscosity."""
        return float(np.clip(0.8 - 0.5 * T + 0.4 * f + 0.1 * f * (1 - T), 0, 1))

    def step(self, action):
        heat_in = action[0] * 0.5 + 0.5   # scale [-1,1] -> [0,1]
        T_new = np.clip(self.alpha_T * T + self.beta_T * heat_in
                        + self.rng.normal(0, self.noise_std), 0, 1)
        reward = (-2.0 * (T_new - self.T_target)**2
                  - 2.0 * (f_new - self.f_target)**2
                  - 0.1 * float(np.sum(action**2)))

The reward penalizes deviation from targets plus action roughness — a standard process control objective.

Dataset: Noisy PID Behavior Policy

The offline dataset is collected from a proportional controller with Gaussian noise (noise_scale=0.3). This produces mixed-quality data — some good trajectories, some poor ones — which is realistic for industrial logs:

def pid_action(state, T_target, f_target):
    T, f, _ = state
    return np.array([
        np.clip(2.0 * (T_target - T), -1, 1),
        np.clip(2.0 * (f_target - f), -1, 1),
    ], dtype=np.float32)

def collect_offline_dataset(n_episodes=400, noise_scale=0.3, seed=0):

Networks

QNetwork takes (state, action) concatenated and returns a scalar value. GaussianPolicy uses the SAC-style stochastic actor with tanh squashing — outputs are guaranteed in [-1, 1], matching the action space:

class QNetwork(nn.Module):
    def __init__(self, state_dim, action_dim, hidden_dim=256):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim + action_dim, hidden_dim), nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),              nn.ReLU(),
            nn.Linear(hidden_dim, 1),
        )
    def forward(self, state, action):
        return self.net(torch.cat([state, action], -1)).squeeze(-1)


class GaussianPolicy(nn.Module):
    def sample(self, state):
        """Returns (action in [-1,1], log_prob) via reparameterization."""
        mean, std = self._dist(state)
        raw    = mean + std * torch.randn_like(mean)
        action = torch.tanh(raw)
        lp     = (torch.distributions.Normal(mean, std).log_prob(raw)
                  - torch.log(1 - action.pow(2) + 1e-6)).sum(-1)
        return action, lp

The log(1 - tanh²(x) + ε) term corrects the log-probability for the Jacobian of the tanh transformation.

The CQL Loss

This is the core of the file. The function takes two Q-networks and their targets (double-Q), the policy, and a batch of transitions:

def compute_cql_loss(Q, Q_other, Q_tgt, Q_other_tgt, policy,
                     states, actions, rewards, next_states, dones,
                     alpha_cql=1.0, alpha_ent=0.1, gamma=0.99, n_samples=10):

    # ── TD target (SAC Bellman backup) ──────────────────────────────────
    with torch.no_grad():
        a_next, lp_next = policy.sample(next_states)
        v_next = (torch.min(Q_tgt(next_states, a_next),
                            Q_other_tgt(next_states, a_next))
                  - alpha_ent * lp_next)
        td_target = rewards + gamma * (1 - dones) * v_next

    q_data  = Q(states, actions)
    td_loss = F.mse_loss(q_data, td_target)

    # ── CQL penalty ──────────────────────────────────────────────────────
    # Expand states for vectorized sampling: (B, D) -> (B*n_samples, D)
    s_rep = states.unsqueeze(1).expand(-1, n_samples, -1).reshape(B * n_samples, -1)

    # Random OOD actions
    a_rand = torch.FloatTensor(B * n_samples, actions.shape[-1]).uniform_(-1, 1)

    # Policy actions (importance-weighted)
    a_pi, lp_pi = policy.sample(s_rep)

    q_rand = Q(s_rep, a_rand).reshape(B, n_samples)
    q_pi   = (Q(s_rep, a_pi) - lp_pi.detach()).reshape(B, n_samples)

    # logsumexp over OOD samples — "push down" term
    logsumexp   = torch.logsumexp(torch.cat([q_rand, q_pi], dim=1), dim=1)
    cql_penalty = (logsumexp - q_data).mean()

    return td_loss + alpha_cql * cql_penalty, {...}

Three things to notice:

Double-Q in the TD target. torch.min(Q_tgt, Q_other_tgt) reduces overestimation in the backup independently of CQL.

Two OOD proposals. Random uniform actions cover the full action space. Policy actions add importance weighting — q_pi - lp_pi approximates $\log \mathbb{E}_\mu[\exp Q / \mu]$.

q_data is the anchor. The TD loss keeps Q(s,a) close to the Bellman target at dataset points. The CQL penalty ensures it stays above the logsumexp of OOD values.

CQLAgent

The agent wraps double-Q + policy + optional automatic alpha tuning via Lagrangian dual gradient descent:

class CQLAgent:
    def update(self, batch):
        s, a, r, s2, d = [x.to(self.device) for x in batch]

        # Update both Q-networks
        l1, i1 = compute_cql_loss(self.Q1, self.Q2, self.Q1_tgt, self.Q2_tgt, ...)
        l2, i2 = compute_cql_loss(self.Q2, self.Q1, self.Q2_tgt, self.Q1_tgt, ...)
        self.q_opt.zero_grad()
        (l1 + l2).backward()
        nn.utils.clip_grad_norm_([...], 1.0)
        self.q_opt.step()

        # Auto alpha: adjust conservatism to keep penalty near target_cql
        if self.auto_alpha:
            pen = (i1['cql_penalty'] + i2['cql_penalty']) / 2
            al  = -self.log_alpha * (pen - self.target_cql)
            self.alpha_opt.zero_grad(); al.backward(); self.alpha_opt.step()

        # Policy: maximize Q - entropy
        a_pi, lp = self.policy.sample(s)
        q_pi = torch.min(self.Q1(s, a_pi), self.Q2(s, a_pi))
        lpi  = (self.alpha_ent * lp - q_pi).mean()
        self.pi_opt.zero_grad(); lpi.backward(); self.pi_opt.step()

        # Soft target update: θ_target ← (1-τ)θ_target + τθ
        for p, pt in zip(self.Q1.parameters(), self.Q1_tgt.parameters()):
            pt.data.mul_(1 - self.tau).add_(self.tau * p.data)

The alpha Hyperparameter

$\alpha$ is the most important hyperparameter in CQL:

$\alpha$ Behavior
$\alpha = 0$ Standard SAC — no conservatism, OOD exploitation
$\alpha = 1.0$ Default starting point
$\alpha = 5$–$10$ Strong conservatism — policy close to $\pi_\beta$
$\alpha \to \infty$ Equivalent to Behavioral Cloning

Automatic tuning uses a Lagrangian: keep the CQL penalty near a target $\tau$. In CQLAgent, set auto_alpha=True and target_cql=-1.0. The dual variable log_alpha is updated to satisfy the constraint:

$$\min_\theta \max_{\alpha \geq 0} \; \mathcal{L}_{TD}(\theta) + \alpha \left( \mathcal{R}_{CQL}(\theta) - \tau \right)$$

What CQL Actually Does: Intuition

Think of the Q-function as a landscape over the action space. Standard Q-learning leaves this landscape unconstrained between data points — it tends to rise due to optimistic generalization.

CQL adds gravity: it pulls the entire landscape down, while TD loss anchors it high at data points. The greedy policy sees high values only at data-like actions and naturally stays in-distribution.

Standard Q:          CQL Q:

  Q ↑    *    *       Q ↑    *    *
     |  / \  /           |  *  \  /
     | /   \/            | /    \/
     |/    /\            |/     /\
     +---------→ a       +----------→ a
     ↑ OOD peaks         ↑ OOD pushed down

Practical Tips

Normalize observations to zero mean, unit variance using dataset statistics. CQL is sensitive to scale — unnormalized states will make the penalty and TD loss imbalanced.

Monitor the CQL penalty (E_μ[Q] - E_D[Q]) during training. It should be positive (OOD values below dataset values) and stable. If it goes negative, increase $\alpha$. If Q-values diverge, increase $\alpha$ or reduce the learning rate.

Use double-Q always. torch.min(Q1, Q2) in the TD target and policy update reduces overestimation independently of CQL — the two mechanisms complement each other.

For multi-regime industrial data (e.g., stopped/slow/fast operating modes): consider adding the regime index as a state feature, or training separate agents per regime. A single CQL agent averaging over regimes will be excessively conservative in each of them.

Limitations

Requires reward labels. Unlike BC, CQL needs (s, a, r, s'). If your dataset only has (s, a, s'), you need to define a reward function separately.

Sensitive to reward scale. Normalize rewards to [-1, 1] or [0, 1]. If rewards are large (thousands), the TD loss dominates and CQL has negligible effect.

Conservative by design. CQL will not extrapolate far beyond the behavior policy. For tasks requiring aggressive improvement over the dataset, model-based methods (Chapter 5) are more appropriate.
