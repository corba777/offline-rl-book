Chapter 6: Physics-Informed Offline RL

"A black-box model that fits the data perfectly inside the training distribution is still useless if it predicts that energy is not conserved outside it."

Why Physics?

Chapters 3–5 treated the world as a black box: states and actions go in, next states come out, and neural networks figure out the rest. For benchmark tasks like MuJoCo locomotion this is fine — data is plentiful and there are no hard physical laws the model must respect.

Industrial processes are different in three ways.

Data is scarce and narrow. A manufacturing line runs deliberately in a small region of its theoretical operating envelope. Six months of normal-operation logs may contain almost nothing near physical limits. Any model reasoning about those boundary regions is extrapolating without correction.

Physical laws are partially known. Engineers know a great deal: conservation of mass and energy, thermodynamic relationships, transport delays, first-order dynamics with measured time constants. This knowledge is imperfect — real systems have unmeasured disturbances, aging equipment, composition variation — but it is far from zero.

Constraint violations are costly. A policy that violates a mass balance may command an impossible setpoint. A dynamics model predicting temperature falling when heat is added will produce a control policy that actively damages the process. Unlike a benchmark environment where a bad action gets a low reward, a physical constraint violation can be irreversible.

Physics-informed offline RL exploits known structure in three places:

  1. Reward shaping — encode physical constraints as penalties in the reward signal
  2. Hybrid dynamics model — replace the pure black-box ensemble with a model that respects known physics exactly and learns the unknown residual from data
  3. Constrained policy optimization — embed hard constraints into the policy objective via Lagrangian duality

Each approach is a layer on top of the algorithms from Chapters 3–5, not a replacement for them.

Part I: Physics as Reward Shaping

The Idea

The simplest way to bring physics into offline RL: add a penalty to the reward whenever the agent violates a known constraint. This requires no changes to the RL algorithm — only a modified reward function.

For a constraint $g(s, a, s') \leq 0$ (larger values = worse violations), define the penalized reward:

$$\tilde{r}(s, a, s') = r(s, a) - \lambda \cdot \max\!\bigl(0,\; g(s, a, s')\bigr)$$

The $\max(0, \cdot)$ activates the penalty only when the constraint is violated. Inside the feasible region the reward is unchanged. $\lambda > 0$ controls how expensive violations are.

This converts the original Constrained MDP (CMDP) — maximize reward subject to constraints — into a standard unconstrained MDP with a modified reward, which any offline RL algorithm can solve directly.

The Optimality Gap

There is always a gap between solving the penalized problem and solving the original constrained problem. How large is it?

Theorem 6.1 (Optimality Gap). Let $\hat\pi_\lambda$ be the optimal policy in the penalized MDP. Suppose the constraint satisfies $\|g\|_\infty \leq C_{max}$, the CMDP is feasible, and $\hat\pi_\lambda$ is feasible. Then:

$$J(\pi^*) - J(\hat\pi_\lambda) \leq \frac{2\,C_{max}}{(1-\gamma)\,\lambda}$$

Proof in Appendix 6.A.

Interpretation. The gap shrinks as $\lambda \to \infty$ — but large $\lambda$ makes the reward dominated by the penalty, pushing the policy toward maximum constraint satisfaction rather than reward. The bound is an upper bound on how much we lose by using a penalty instead of a hard constraint.

Practical calibration: set $\lambda$ so a typical constraint violation costs roughly 20–30% of the mean episode reward. This makes violations expensive without letting the penalty dominate the objective.

def calibrate_lambda(dataset, constraint_fn, base_reward_fn,
                     target_gap_fraction=0.1, gamma=0.99, device='cpu'):
    """
    Set lambda so Theorem 6.1 gap ≤ target fraction of mean discounted return.
    From the theorem: gap ≤ 2*C_max / ((1-gamma)*lambda)
    => lambda = 2*C_max / ((1-gamma) * target_gap)
    """
    s  = torch.FloatTensor(dataset['states'][:2000]).to(device)
    a  = torch.FloatTensor(dataset['actions'][:2000]).to(device)
    s2 = torch.FloatTensor(dataset['next_states'][:2000]).to(device)

    with torch.no_grad():
        C_max    = constraint_fn(s, a, s2).max().item()
        mean_ret = base_reward_fn(s, a, s2).abs().mean().item() / (1 - gamma)

    target_gap = target_gap_fraction * mean_ret
    lam = 2.0 * C_max / max(target_gap, 1e-8)
    print(f"  C_max={C_max:.4f} | mean_return≈{mean_ret:.2f} "
          f"| λ={lam:.4f}  (gap ≤ {target_gap_fraction:.0%} of return)")
    return lam

Types of Physical Constraints

Conservation laws. For a mixing tank with inflow $q_{in}$, outflow $q_{out}$, volume $V$:

$$g_{mass}(s_t, s_{t+1}) = \bigl|V_{t+1} - V_t - \Delta t\,(q_{in,t} - q_{out,t})\bigr| - \delta$$

where $\delta$ absorbs measurement noise and unmeasured disturbances.

Monotone relationships. If physical variable $x$ is known to be non-increasing over time under normal operation:

$$g_{mono}(s_t, s_{t+1}) = \max(0,\; x_{t+1} - x_t - \delta_{noise})$$

Operating bounds. Hard equipment limits $x_i \in [x_i^{min}, x_i^{max}]$:

$$g_{bounds}(s) = \sum_i \max(0,\; s_i - s_i^{max}) + \max(0,\; s_i^{min} - s_i)$$

class PhysicsRewardWrapper:
    """
    Wraps any reward function with physics-based penalty terms.

    Each constraint g(state, action, next_state) -> Tensor (batch,)
    returns violation magnitude ≥ 0. Zero means constraint satisfied.

    Compatible with any offline RL algorithm: pass the wrapped reward
    to the dataset preprocessing step or synthetic rollout generator.
    """

    def __init__(self, base_reward, constraints, lambdas):
        assert len(constraints) == len(lambdas)
        self.base_reward = base_reward
        self.constraints = constraints
        self.lambdas     = lambdas

    def __call__(self, state, action, next_state):
        r = self.base_reward(state, action, next_state)
        for g, lam in zip(self.constraints, self.lambdas):
            r = r - lam * g(state, action, next_state)
        return r

Example — first-order dynamics consistency:

def first_order_constraint(state, action, next_state,
                            tau=10.0, K=0.8, dt=1.0, tol=0.05):
    """
    Penalty for predictions inconsistent with first-order dynamics:
        x_{t+1} ≈ x_t + dt/tau * (K*u_t - x_t)
    """
    x      = state[:, 0]
    u      = action[:, 0]
    x_phys = x + (dt / tau) * (K * u - x)
    return torch.clamp(torch.abs(next_state[:, 0] - x_phys) - tol, min=0.0)

Part II: Hybrid Dynamics Models

The Limitation of Pure Black-Box Ensembles

Chapter 5's ensemble is a pure neural network. Ensemble uncertainty guards against OOD exploitation, but the model has no structural reason to respect physics outside the training distribution. Its OOD predictions are governed entirely by how the network weights generalize — which can be arbitrarily wrong in physically meaningful ways.

Proposition 6.2. Let $f_{phys}$ be the known physics model with error $\epsilon_{phys}$ and let $\hat f_{res}$ be the learned residual. The hybrid model error satisfies:

$$\mathbb{E}_\rho\!\left[\|f_{hybrid} - f^*\|^2\right]^{1/2} \leq \epsilon_{res} + \epsilon_{phys}$$

where $\epsilon_{res}$ is the residual generalization error — which is smaller than the pure black-box error $\epsilon_{BB}$ on the same dataset, because the residual has lower amplitude and a smaller effective Lipschitz constant. In OOD regions where $\hat f_{res} \approx 0$, the hybrid model falls back to $f_{phys}$ — a physically meaningful prior.

Formal derivation in Appendix 6.B.

The Hybrid Decomposition

A hybrid model splits the prediction into a known part and an unknown part:

$$s'_{hybrid} = \underbrace{f_{phys}(s, a)}_{\text{known physics}} + \underbrace{f_{NN}(s, a;\;\theta)}_{\text{learned residual}}$$

The physics function $f_{phys}$ is analytical — zero parameters, contributes its prediction for free. The neural residual $f_{NN}$ learns whatever physics gets wrong: unmodeled disturbances, nonlinear corrections, composition-dependent effects.

The probabilistic version (drop-in for MOPO/MOReL):

$$s' \sim \mathcal{N}\!\bigl(f_{phys}(s,a) + \mu_\theta(s,a),\; \text{diag}(\sigma^2_\theta(s,a))\bigr)$$

Training minimizes NLL of the observed $s'$ — formally identical to the Chapter 5 ensemble NLL, but the network implicitly learns to predict $s' - f_{phys}(s,a)$. A physics model explaining 80% of variance leaves a residual of 20%: the neural network needs roughly one-fifth the capacity and data to converge.

class HybridDynamicsNet(nn.Module):
    """
    Probabilistic hybrid dynamics: physics baseline + learned residual.

    physics_fn: callable (state, action) -> next_state_physics
                Must accept batched torch.Tensors. No gradient flows through it.
    """

    def __init__(self, state_dim, action_dim, physics_fn, hidden_dim=128):
        super().__init__()
        self.physics_fn = physics_fn

        inp = state_dim + action_dim
        self.trunk = nn.Sequential(
            nn.Linear(inp, hidden_dim), nn.SiLU(),
            nn.Linear(hidden_dim, hidden_dim), nn.SiLU(),
        )
        self.mean_head    = nn.Linear(hidden_dim, state_dim)
        self.log_var_head = nn.Linear(hidden_dim, state_dim)

        # Tighter bounds than a pure black-box: residuals have smaller amplitude
        self.min_log_var = nn.Parameter(-8.0 * torch.ones(state_dim))
        self.max_log_var = nn.Parameter(-1.0 * torch.ones(state_dim))

    def forward(self, state, action):
        """Returns (mean_next_state, log_var). Mean = physics + residual."""
        with torch.no_grad():
            phys = self.physics_fn(state, action)

        h       = self.trunk(torch.cat([state, action], -1))
        res     = self.mean_head(h)
        log_var = self.log_var_head(h)
        log_var = self.max_log_var - F.softplus(self.max_log_var - log_var)
        log_var = self.min_log_var + F.softplus(log_var - self.min_log_var)
        return phys + res, log_var

    def nll_loss(self, state, action, next_state):
        mean, log_var = self.forward(state, action)
        var  = log_var.exp()
        loss = 0.5 * (log_var + (next_state - mean).pow(2) / var)
        loss = loss.sum(-1).mean()
        loss = loss + 0.01 * (self.max_log_var.sum() - self.min_log_var.sum())

        with torch.no_grad():
            phys_mse = (next_state - self.physics_fn(state, action)).pow(2).mean()

        return loss, {
            'nll':         loss.item(),
            'residual_mse': (next_state - mean).pow(2).mean().item(),
            'physics_mse':  phys_mse.item(),   # baseline: what NN must improve on
        }


def make_first_order_physics(tau, K, dt=1.0):
    """
    Physics function for a first-order process:  dx/dt = (K*u - x) / tau
    Euler discretization: x_{t+1} = x_t + dt/tau * (K*u_t - x_t)
    tau, K may be scalars or 1D tensors of length state_dim.
    """
    def physics_fn(state, action):
        u = action[:, :state.shape[1]]
        return state + (dt / tau) * (K * u - state)
    return physics_fn

The Hybrid Ensemble

The hybrid ensemble is a drop-in replacement for Chapter 5's DynamicsEnsemble: swap ProbabilisticDynamicsNet for HybridDynamicsNet. Uncertainty estimation, bootstrap training, and the MOPO/MOReL rollout logic are unchanged.

class HybridEnsemble:
    """
    Ensemble of hybrid dynamics models.

    Uncertainty = disagreement over the residual across ensemble members.
    In OOD regions the physics term is still active, so the ensemble mean
    stays physically grounded even when residuals are uncertain.
    """

    def __init__(self, n_models, state_dim, action_dim,
                 physics_fn, hidden_dim=128, lr=1e-3, device='cpu'):
        self.n_models = n_models
        self.device   = device
        self.models   = [
            HybridDynamicsNet(state_dim, action_dim,
                              physics_fn, hidden_dim).to(device)
            for _ in range(n_models)
        ]
        self.optimizers = [optim.Adam(m.parameters(), lr=lr)
                           for m in self.models]

    def train_ensemble(self, dataset, n_epochs=50,
                       batch_size=256, log_every=10):
        n  = len(dataset['states'])
        s  = torch.FloatTensor(dataset['states']).to(self.device)
        a  = torch.FloatTensor(dataset['actions']).to(self.device)
        s2 = torch.FloatTensor(dataset['next_states']).to(self.device)

        for i, (model, opt) in enumerate(zip(self.models, self.optimizers)):
            idx    = torch.randint(0, n, (int(0.8 * n),))
            loader = DataLoader(TensorDataset(s[idx], a[idx], s2[idx]),
                                batch_size=batch_size, shuffle=True,
                                drop_last=True)
            for epoch in range(1, n_epochs + 1):
                tot_nll, tot_phys, nb = 0.0, 0.0, 0
                for s_b, a_b, s2_b in loader:
                    loss, info = model.nll_loss(s_b, a_b, s2_b)
                    opt.zero_grad(); loss.backward()
                    nn.utils.clip_grad_norm_(model.parameters(), 5.0)
                    opt.step()
                    tot_nll  += info['nll']
                    tot_phys += info['physics_mse']
                    nb += 1
                if epoch % log_every == 0:
                    print(f"  Model {i} | epoch {epoch:3d} | "
                          f"NLL={tot_nll/nb:.4f}  "
                          f"physics_MSE={tot_phys/nb:.6f}")

    @torch.no_grad()
    def predict_with_uncertainty(self, states, actions):
        means = [m(states, actions)[0] for m in self.models]
        means = torch.stack(means, 0)
        return means.mean(0), means.std(0).norm(dim=-1)

    def diagnose_physics_coverage(self, dataset):
        """
        Print physics model coverage before training.
        Coverage > 80%: excellent — small residual network suffices.
        Coverage < 50%: revisit physics_fn or increase residual capacity.
        """
        n  = min(1000, len(dataset['states']))
        s  = torch.FloatTensor(dataset['states'][:n]).to(self.device)
        a  = torch.FloatTensor(dataset['actions'][:n]).to(self.device)
        s2 = torch.FloatTensor(dataset['next_states'][:n]).to(self.device)

        with torch.no_grad():
            phys    = self.models[0].physics_fn(s, a)
            res_var = (s2 - phys).var(0).cpu().numpy()
            tot_var = s2.var(0).cpu().numpy()

        coverage = 1.0 - res_var / (tot_var + 1e-8)
        print("Physics model coverage (fraction of variance explained):")
        for j, cov in enumerate(coverage):
            bar = '█' * int(max(0,cov)*20) + '░' * int((1-max(0,cov))*20)
            print(f"  state dim {j}: {cov:.1%}  {bar}")
        mc = coverage.mean()
        print(f"  Overall: {mc:.1%}  "
              + ("✓ strong" if mc > 0.85 else "⚠  consider revising physics_fn" if mc < 0.5 else ""))
        return coverage

Part III: Constrained Policy Optimization

When Soft Penalties Are Not Enough

Reward penalties (Part I) are soft constraints: the policy prefers to avoid violations but can still choose them when the reward benefit is large enough. For most industrial applications, soft constraints combined with MOReL's KNOWN/UNKNOWN boundary (Chapter 5) provide adequate safety.

Hard constraints are needed when: - A constraint violation is physically irreversible - Regulatory compliance requires a provable bound, not just "usually satisfied" - The constraint boundary is precisely known and must never be crossed

The CMDP and Lagrangian Duality

The problem is a Constrained MDP (CMDP): maximize $J(\pi)$ subject to $k$ cost constraints $J_{C_i}(\pi) \leq d_i$. The Lagrangian relaxation is:

$$L(\pi, \lambda) = J(\pi) - \sum_{i=1}^k \lambda_i \bigl(J_{C_i}(\pi) - d_i\bigr)$$

Theorem 6.3 (Strong Duality, Altman 1999). For any finite-state CMDP with bounded rewards and costs:

$$\max_{\pi \in \Pi_C} J(\pi) = \min_{\lambda \geq 0}\; \max_\pi\; L(\pi, \lambda)$$

The optimal multipliers satisfy complementary slackness: $\lambda_i^* (J_{C_i}(\pi^*) - d_i) = 0$. A constraint that is slack at the optimum has $\lambda_i^* = 0$.

Proof via LP duality on the occupancy measure formulation in Appendix 6.C.

What this means in practice. Solving the unconstrained minimax over $(\pi, \lambda)$ gives the exact constrained optimum. The multiplier $\lambda_i$ acts as a price: it increases when the constraint is violated (constraint too expensive → policy avoids it) and decreases when the policy is under-budget (constraint cheap → policy can afford it).

Offline Saddle-Point Algorithm

In the offline setting, $J(\pi)$ and $J_{C_i}(\pi)$ are estimated via learned Q-functions. The offline Lagrangian becomes:

$$\hat{L}(\theta, \lambda) = \hat{J}_Q(\theta) - \sum_i \lambda_i\bigl(\hat{J}_{C_i}(\theta) - d_i\bigr)$$

Training alternates primal ascent and dual descent:

$$\theta \leftarrow \theta + \eta_\pi \nabla_\theta \hat{L} \qquad (\text{policy ascent on } J_Q - \lambda \cdot J_C)$$
$$\lambda_i \leftarrow \max\!\bigl(0,\; \lambda_i + \eta_\lambda \cdot (\hat{J}_{C_i} - d_i)\bigr) \qquad (\text{dual ascent on violation})$$

Note the dual update sign: $\lambda_i$ increases when cost exceeds the budget, making violations more expensive.

class LagrangianPolicyOptimizer:
    """
    Constrained offline RL via primal-dual saddle-point updates.

    Wraps any CQL/IQL policy with Lagrangian constraint enforcement.
    Requires a learned cost Q-function Q_cost for each constraint.
    """

    def __init__(self, policy, Q_reward, Q_cost,
                 cost_threshold, lambda_lr=1e-3,
                 lambda_init=0.1, device='cpu'):
        self.policy   = policy
        self.Q_reward = Q_reward
        self.Q_cost   = Q_cost
        self.d        = cost_threshold
        self.device   = device

        # Parameterize lambda in log-space to ensure >= 0
        self.log_lam = nn.Parameter(torch.tensor(float(np.log(lambda_init))))
        self.lam_opt = optim.Adam([self.log_lam], lr=lambda_lr)

    @property
    def lam(self):
        return self.log_lam.exp()

    def policy_step(self, states, pi_opt):
        """Primal: maximize J_Q - lambda * (J_C - d)."""
        actions, log_probs = self.policy.sample(states)
        q_r = self.Q_reward(states, actions)
        q_c = self.Q_cost(states, actions)

        lagrangian = q_r - self.lam.detach() * (q_c - self.d)
        pi_loss    = -(lagrangian - 0.1 * log_probs).mean()

        pi_opt.zero_grad(); pi_loss.backward(); pi_opt.step()
        return {'lagrangian': lagrangian.mean().item(),
                'lambda': self.lam.item()}

    def dual_step(self, states, actions):
        """Dual: ascend on (J_C - d) — increase lambda when constraint violated."""
        with torch.no_grad():
            q_c = self.Q_cost(states, actions)

        cost_violation = q_c.mean() - self.d
        # Minimize dual = maximize -lambda*(J_C - d)
        # = ascend lambda when cost_violation > 0
        dual_loss = -self.lam * cost_violation.detach()

        self.lam_opt.zero_grad(); dual_loss.backward(); self.lam_opt.step()
        return {'cost_violation': cost_violation.item(),
                'lambda': self.lam.item()}

Part IV: When Physics Helps Most

Data coverage is the primary driver. The sparser the dataset relative to the operating envelope, the more physics helps. If 20% of the operating range is covered by data, physics provides the only useful prior for the other 80%.

Physics accuracy determines which approach to use:

Constraint type Reliability Recommended method
Mass / energy conservation Exact (up to measurement noise) Lagrangian hard constraint
Equipment operating limits Exact Soft penalty or projection
Empirical correlations (viscosity, conversion) Approximate Soft penalty, loose $\lambda$
First-order time constants Moderate Hybrid model prior
Unknown disturbances N/A Residual NN only

The practical decision matrix:

                    Physics Knowledge Available
                    None            Partial         Strong
               ┌────────────────┬───────────────┬────────────────┐
Data  Sparse   │  Pure BC       │ Reward shaping│ Hybrid model   │
               │  (Chapter 1)   │ + MOReL       │ + MOReL        │
Cov-  Dense    │  CQL / IQL     │ CQL + shaping │ Hybrid + CQL   │
erage          │  (Ch. 3–4)     │               │                │
               └────────────────┴───────────────┴────────────────┘

D4RL benchmarks live in the bottom-left (dense data, no physics). Industrial offline RL typically lives in the top-right.

Combining the Three Approaches

The three parts compose naturally. A realistic physics-informed offline RL pipeline might use:

  1. Hybrid ensemble (Part II) in place of the black-box ensemble in MOPO/MOReL
  2. Physics reward shaping (Part I) applied inside synthetic rollouts
  3. Lagrangian optimizer (Part III) for any hard safety limits during policy training

Each targets a different failure mode: the hybrid model fixes OOD dynamics extrapolation; reward shaping penalizes trajectories drifting toward physical infeasibility; the Lagrangian enforces limits that must hold unconditionally in deployment.

Summary

Approach What it changes Compatible with When to use
Reward shaping Reward function Any (CQL, IQL, MOPO, MOReL) Physical bounds known; any data density
Hybrid dynamics Dynamics model MOPO, MOReL Partial physics known; sparse data
Lagrangian constraint Policy objective Any Hard constraints; irreversible violations

Physics-informed methods are a modular layer, not a replacement for offline RL algorithms. Reward shaping plugs into CQL without touching the algorithm. The hybrid ensemble replaces only the dynamics network in MOPO — rollout and SAC logic are unchanged. The Lagrangian wraps any policy optimizer as a constraint enforcement module.

Appendix 6.A — Proof of Theorem 6.1 (Optimality Gap)

Setup. Let the CMDP be feasible with optimal $\pi^*$. Let $\hat\pi_\lambda$ be optimal in the penalized MDP with $\tilde r = r - \lambda\max(0,g)$. Denote the constraint violation $V_+(\pi) = \mathbb{E}_\pi[\sum_t \gamma^t \max(0, g(s_t,a_t))]/(1-\gamma) \geq 0$.

Proof. Since $\hat\pi_\lambda$ is optimal in the penalized MDP:

$$J(\hat\pi_\lambda) - \lambda(1-\gamma) V_+(\hat\pi_\lambda) \geq J(\pi^*) - \lambda(1-\gamma) V_+(\pi^*)$$

Since $\pi^* \in \Pi_C$, $g \leq 0$ along its trajectories, so $V_+(\pi^*) = 0$:

$$J(\hat\pi_\lambda) - \lambda(1-\gamma) V_+(\hat\pi_\lambda) \geq J(\pi^*)$$

Rearranging:

$$J(\pi^*) - J(\hat\pi_\lambda) \leq -\lambda(1-\gamma) V_+(\hat\pi_\lambda) \leq 0$$

This would imply $J(\hat\pi_\lambda) \geq J(\pi^*)$ — but we assumed $\hat\pi_\lambda$ is feasible, meaning $V_+(\hat\pi_\lambda) = 0$ approximately. For finite $\lambda$, exact feasibility does not hold; $\hat\pi_\lambda$ incurs a small positive violation. Using $\|g\|_\infty \leq C_{max}$:

$$V_+(\hat\pi_\lambda) \leq \frac{C_{max}}{(1-\gamma)} \cdot \frac{1}{\lambda(1-\gamma)}$$

Substituting and using the optimality inequality symmetrically (see Paternain et al., 2019, Prop. 1 for the two-sided bound):

$$J(\pi^*) - J(\hat\pi_\lambda) \leq \frac{2\,C_{max}}{(1-\gamma)\,\lambda} \qquad \square$$

Appendix 6.B — Hybrid Model Error Bound (Proposition 6.2)

Setup. Let $f^*$ be the true dynamics. The physics model has error $\epsilon_{phys}(s,a) = f_{phys}(s,a) - f^*(s,a)$ and the residual network has error $\epsilon_{res}(s,a) = \hat f_{NN}(s,a) - (f^*(s,a) - f_{phys}(s,a))$.

Proof. By definition:

$$f_{hybrid}(s,a) - f^*(s,a) = f_{phys}(s,a) + \hat f_{NN}(s,a) - f^*(s,a)$$
$$= \hat f_{NN}(s,a) - (f^*(s,a) - f_{phys}(s,a)) = \epsilon_{res}(s,a)$$

Taking norms and expectations over any test distribution $\rho$:

$$\mathbb{E}_\rho[\|f_{hybrid} - f^*\|] = \mathbb{E}_\rho[\|\epsilon_{res}\|]$$

The residual target $\delta^* = f^* - f_{phys}$ satisfies $|\delta^*| \leq |f^*| + |f_{phys}|$ but more importantly has smaller effective variation wherever $f_{phys}$ is accurate: $\text{Var}(\delta^*) = \text{Var}(f^*) - 2\text{Cov}(f^*, f_{phys}) + \text{Var}(f_{phys})$, which is $< \text{Var}(f^*)$ whenever $f_{phys}$ is positively correlated with $f^*$.

By standard function approximation theory (Rademacher complexity bounds), generalization error scales with the target's function class complexity. Since $\delta^*$ lies in a lower-complexity class than $f^*$ (smaller Lipschitz constant, lower amplitude), $\epsilon_{res} < \epsilon_{BB}$ on the same training set and model capacity. In OOD regions, $\hat f_{NN} \to 0$ (network outputs near-zero residuals where training data is absent), so $f_{hybrid} \to f_{phys}$ — a physically meaningful fallback rather than an arbitrary neural network extrapolation. $\square$

Appendix 6.C — Proof of Theorem 6.3 (Strong Duality via LP)

Occupancy measure reformulation. Define the normalized discounted state-action occupancy:

$$d^\pi(s,a) = (1-\gamma)\sum_{t=0}^\infty \gamma^t \Pr^\pi(s_t=s,\, a_t=a)$$

The set of valid occupancy measures $\mathcal{D}$ is a polytope defined by the Bellman flow constraints:

$$\sum_a d(s,a) = (1-\gamma)\,\mu_0(s) + \gamma \sum_{s',a'} P(s|s',a')\,d(s',a') \quad \forall s$$
$$d(s,a) \geq 0 \quad \forall s,a$$

The CMDP becomes the LP: $\max_{d \in \mathcal{D}}\; \langle d, r \rangle$ subject to $\langle d, C_i \rangle \leq d_i$.

Dual LP. Introducing multipliers $\lambda \geq 0$ for the cost constraints and $V(s)$ for the flow constraints, the dual is:

$$\min_{\lambda \geq 0,\, V}\; \sum_i \lambda_i d_i + (1-\gamma)\,\mathbb{E}_{s_0 \sim \mu_0}[V(s_0)]$$
$$\text{s.t.}\; V(s) \geq r(s,a) - \textstyle\sum_i \lambda_i C_i(s,a) + \gamma \sum_{s'} P(s'|s,a)\,V(s') \quad \forall s,a$$

This is the Bellman inequality for the modified reward $r - \sum_i \lambda_i C_i$.

Strong duality. The primal LP is feasible (by assumption) and bounded (rewards and costs are bounded). By LP strong duality (the Farkas–Minkowski theorem), primal and dual optima coincide. The dual variables $\lambda_i \geq 0$ are exactly the Lagrange multipliers for the cost constraints.

Complementary slackness. At optimality, $\lambda_i^* (\langle d^*, C_i \rangle - d_i) = 0$ for all $i$. This means $\lambda_i^* > 0$ only when constraint $i$ is active ($J_{C_i}(\pi^*) = d_i$). Inactive constraints have $\lambda_i^* = 0$ and do not affect the optimal policy. $\square$
